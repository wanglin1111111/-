# 写作辅助技能

全面的写作辅助工具，从构思到成稿提供全程支持。

## 核心功能

### 1. 写作风格分析

分析文本的风格特点，包括：
- **语言风格**：正式/非正式、学术/通俗、简洁/繁复
- **情感基调**：积极/消极、客观/主观、热情/冷静
- **目标受众**：专业读者、大众读者、特定群体
- **修辞手法**：比喻、排比、反问等

### 2. 语法纠错检查

全面检查文本中的语法问题：
- **拼写错误**：错别字、标点符号
- **语法错误**：主谓一致、时态错误、句式杂糅
- **用词不当**：近义词混淆、搭配错误
- **逻辑问题**：前后矛盾、指代不明

### 3. 段落结构优化

优化文章的整体结构和段落组织：
- **逻辑结构**：开头-主体-结尾的完整性
- **段落衔接**：过渡自然、层次分明
- **内容密度**：详略得当、重点突出
- **可读性**：句子长度、段落长度优化

### 4. 引用资料查找

辅助查找和整理参考资料：
- **学术文献**：论文、期刊、研究报告
- **权威来源**：官方数据、行业报告
- **相关案例**：实例、数据、统计
- **引用格式**：APA、MLA、Chicago等

## 工作流程

### 写作前：构思与规划

```
1. 明确写作目标
   ├── 写作目的：告知/说服/娱乐/记录
   ├── 目标读者：专业/大众/特定群体
   └── 预期效果：读者应该获得什么

2. 确定内容框架
   ├── 核心观点：文章要表达什么
   ├── 支撑论据：用什么来支持观点
   └── 结构大纲：如何组织内容

3. 收集参考资料
   ├── 权威数据：统计数据、研究报告
   ├── 案例素材：实例、故事、引用
   └── 背景知识：必要的概念解释
```

### 写作中：实时辅助

```
1. 风格一致性检查
   ├── 语气是否统一
   ├── 用词是否恰当
   └── 表达是否清晰

2. 语法即时纠错
   ├── 拼写检查
   ├── 语法验证
   └── 标点规范

3. 结构优化建议
   ├── 段落划分
   ├── 过渡衔接
   └── 重点突出
```

### 写作后：润色与完善

```
1. 全文风格分析
   ├── 整体风格评估
   ├── 改进建议
   └── 读者适配度

2. 深度语法检查
   ├── 复杂句式分析
   ├── 逻辑连贯性
   └── 表达准确性

3. 最终优化
   ├── 标题优化
   ├── 开头结尾强化
   └── 细节打磨
```

## 工具使用指南

### analyze-style: 分析写作风格

**适用场景：**
- 不确定文本风格是否合适
- 需要调整语言风格
- 分析他人作品学习风格

**使用示例：**
```
"分析这段文字的风格特点"
"这段文字适合什么类型的读者？"
"这段文字的语言风格是正式还是非正式？"
```

**输出内容：**
- 语言风格标签
- 情感基调分析
- 目标受众判断
- 修辞手法识别
- 改进建议

### check-grammar: 语法检查

**适用场景：**
- 完成初稿后的检查
- 重要文档发布前
- 学习语法规则

**使用示例：**
```
"检查这篇文章的语法错误"
"这段文字有没有用词不当的地方？"
"帮我检查标点符号使用是否正确"
```

**输出内容：**
- 错误列表（位置、类型、建议）
- 修改后的文本
- 语法规则说明

### optimize-structure: 结构优化

**适用场景：**
- 文章逻辑不清晰
- 段落组织混乱
- 需要提升可读性

**使用示例：**
```
"优化这个段落的结构"
"这篇文章的逻辑结构有什么问题？"
"如何让这段文字更有条理？"
```

**输出内容：**
- 结构问题诊断
- 优化方案
- 改写示例

### find-references: 查找参考资料

**适用场景：**
- 需要数据支撑论点
- 寻找权威引用
- 了解研究现状

**使用示例：**
```
"找一些关于人工智能的参考资料"
"有没有关于这个主题的最新研究？"
"帮我找几个相关的案例"
```

**输出内容：**
- 相关资料列表
- 关键信息摘要
- 引用格式建议

## 最佳实践

### 写作风格建议

| 文体类型 | 风格特点 | 注意事项 |
|----------|----------|----------|
| **学术论文** | 客观、严谨、逻辑清晰 | 避免主观臆断，注重证据支撑 |
| **商业报告** | 简洁、专业、数据驱动 | 突出重点，避免冗长 |
| **创意写作** | 生动、形象、情感丰富 | 保持风格一致，避免过度修饰 |
| **技术文档** | 准确、清晰、步骤明确 | 术语一致，示例充分 |
| **自媒体** | 亲切、有趣、易于理解 | 了解受众，互动性强 |

### 常见写作问题及解决方案

#### 问题1：内容空洞

**表现：**
- 泛泛而谈，缺乏具体信息
- 重复表达，没有新内容
- 缺乏数据或案例支撑

**解决方案：**
- 添加具体数据和事实
- 引用权威来源
- 使用具体案例说明

#### 问题2：逻辑混乱

**表现：**
- 观点跳跃，缺乏过渡
- 前后矛盾，无法自洽
- 结构松散，重点不突出

**解决方案：**
- 使用"首先-其次-最后"等过渡词
- 每段一个中心思想
- 开头明确主旨，结尾总结升华

#### 问题3：语言平淡

**表现：**
- 句式单一，缺乏变化
- 用词普通，没有亮点
- 缺乏修辞，不够生动

**解决方案：**
- 长短句结合
- 使用具体动词和形容词
- 适当运用修辞手法

#### 问题4：语法错误

**表现：**
- 错别字、标点错误
- 主谓不一致
- 时态混乱

**解决方案：**
- 完成初稿后专门检查语法
- 大声朗读，发现不通顺的地方
- 让他人帮忙审阅

### 不同文体的写作要点

**学术论文：**
- 文献综述要全面
- 研究方法要详细
- 数据分析要严谨
- 结论要有创新性

**商业报告：**
- 执行摘要要精炼
- 数据分析要直观
- 建议要可操作
- 风险评估要全面

**创意写作：**
- 人物要立体
- 情节要紧凑
- 对话要自然
- 场景要生动
